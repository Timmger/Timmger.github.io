《Git漫游指南》- Git命令

1. 版本控制

1.1 什么是版本控制

版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统。版本控制系统(VCS)，在需要的时候可以将文件回溯到之前的状态，还可以显示当前版本与之前版本文件改动的细节。

1.2 版本控制系统类别

本地版本控制系统

RCS

原理：在硬盘上保存补丁集（文件修订前后变化），通过应用补丁集，可以计算出各个版本文件内容。
缺点：不可协同工作，风险集中

集中的版本控制系统（CVCS）

CVS、Subversion(SVN)、Perforce

原理：由单一集中管理的服务器保存所有的文件修订版本，协同工作通过客户端连接，取出最新文件或提交更新。
优点：增加了项目的透明度，管理员权限集中，维护更容易
缺点：中央服务器风险集中

分布式版本控制系统（DVCS）

Git、Mercurial、Bazaar、Darcs

原理：每个客户端都将代码仓库完整克隆，若发生故障，可由任一镜像的本地仓库恢复。并且，这类系统可以指定若干不同的远端代码仓库进行交互
优点：支持非线性开发模式、完全分布式、高速且适用大型项目（本地执行）、风险趋零。

1.3 Git工作流程

  1. 在工作目录中修改(modified)文件。

  2. 暂存(staged)文件，将文件的快照放入暂存区域。

  3. 提交(committed)更新，找到暂存区域的文件，将快照永久性存储到 Git 仓库目录。

2. 命令行

2.1 初次配置

安装：略

设置用户名和邮件地址

$ git config –global user.name “timmger”
$ git config –global user.email timmgerable@gmail.com

2.2 仓库

仓库(repository), 版本控制中最基本的模块，存储项目积攒的所有版本和元数据。
“本地” 仓库（local repository）: 放置在本地计算机，只有本人有权操作改动该仓库。
“远程” 仓库（remote repository）：位于远程服务器，由“.git”仓库目录组成，团队可进行数据共享与交换。

初始化Git仓库

$ git init：创建一个.git的子目录

追踪文件并提交

$ git add *.c
$ git add LICENSE
$ git commit -m ‘initial project version’

克隆现有仓库

git clone [url]

Git 支持多种数据传输协议，如https://协议，git://协议或SSH传输协议（user@ipaddress:path/to/repo.git）

2.3 记录每次更新

git status: 记录当前的文件状态

显示分支”master”, 没有出现未追踪的新文件

创建README文件

追踪README文件

git add README

README已被追踪，处于暂存状态

暂存已修改文件

git add modifiedfile

简化status输出

??: untracked
M: modified
A: staged
MM: (left)modified,staged, (right)modified,unstaged, 表示修改提交后，又被修改

忽略文件

支持简化正则表达式

如Java.gitignore

更多见: https://github.com/github/gitignore

查看当前文件与暂存区域的区别

git diff

显示暂存修改后的区别

git diff –staged

提交更新

git commit
git commit -m “some information”

跳过暂存

git commit -a

移除文件

git rm: 从暂存区域移除
git rm -f: 强制删除

移动文件

git mv

2.4 查看提交历史

git log
git log -p -2: 用来显示最近两次提交的内容差异
git log –stat: 查看每次提交的简略的统计信息
git log –pretty=oneline: 提交一行显示，–pretty还有short,full,fuller,format选项
git log –pretty=format:”%h - %an, %ar : %s: 定制记录格式
git log –pretty=format:”%h %s” –graph: 用ASCII字符串来形象地展示你的分支、合并历史

format选项
选项 说明
%h 提交对象的简短哈希字串
%an 作者（author）的名字
%ae 作者的电子邮件地址
%ad 作者修订日期
%ar 作者修订日期，按多久以前的方式显示
%cn 提交者（committer）的名字
%s 提交说明
限制 git log 输出的选项
选项 说明
-(n) 仅显示最近的 n 条提交，如-2
--since, --after 仅显示指定时间之后的提交，如--since="2008-10-01"
--until, --before 仅显示指定时间之前的提交
--author 仅显示指定作者相关的提交，如 --author=timmger
--grep 仅显示含指定关键字的提交
--committer 仅显示指定提交者相关的提交
-S 仅显示添加或移除了某个关键字的提交

2.5 撤销操作

重新提交

git commit –amend : 重新提交暂存区域文件

取消暂存文件

git reset HEAD file

撤销对文件的修改

git checkout – file

2.6 远程仓库的使用

显示远程仓库

指定选项 -v，会显示需要读写远程仓库使用的 Git 保存的简写与其对应的 URL

添加远程仓库

git remote add <shortname> url

指定pd代替整个url, 使用git fetch pd可拉取

从远程仓库抓取和拉取

git fetch [remote-name]: 拉取本地没有的数据，不会与当前工作合并

git pull: 抓取并合并

推送到远程仓库

git push [remote-name] [branch-name]

git push origin master: ：把master分支提交到origin服务器

查看远程仓库

git remote show [remote-name]

远程仓库的移除与重命名

git remote rename

git remote rm

2.7 打标签

列出标签

创建附注标签

附注标签其中包含打标签者的名字、电子邮件地址、日期时间；还有GNU Privacy Guard （GPG）签名与验证。

git tag -a v1.4 -m ‘my version 1.4’

创建轻量标签

轻量标签，没有保存任何其他信息。

后期打标签

给”updated rakefile”打标签，提交部分校验和

git tag -a v1.2 9fceb02

显示推送标签

git push origin v1.4: 推送v1.4
git push origin –tags: 一次性推送

删除标签

git tag -d v1.4-lw

更新远程仓库

git push origin :refs/tags/v1.4-lw

2.8 Git别名

通过 git config 文件为每一个命令设置一个别名

设置取消暂存别名

git config –global alias.unstage ‘reset HEAD –’

设置最后一次提交别名

git config –global alias.last ‘log -1 HEAD’
