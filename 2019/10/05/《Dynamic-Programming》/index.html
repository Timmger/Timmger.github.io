《Dynamic Programming》(一)

发表于 2019-10-05 | 分类于 Algorithms | | 阅读次数:
字数统计: 1.9k | 阅读时长 ≈ 8

1. 概述

动态规划主要应用于纯递归问题的优化。只要遇到对相同输入的重复调用时，我们都可以使用动态规划来优化它，主要的思路就是简化子问题结果的存储,这样当之后求解子问题时，我们就不用重复计算。
动态规划算法可以将时间复杂度从指数时间降到多项式时间，通常有以下4个步骤来设计：

  1. 刻画一个最优解的结构特征
  2. 递归地定义最优解的值
  3. 计算最优解的值，通常采用自底向上的方法
  4. 利用计算出的信息构造一个最优解

2. 最长递增子序列

问题描述：
在最长子序列中，输入是一串数列 $a_1$, … , $a_n$。子序列是这些数的任意子集，形如$a_{i_1},a_{i_2},{…} ,a_{i_k}$ ,其中 $1\le{i_1}\lt{i_2}\lt{…}\lt{i_k}\le{n}$，递增子序列是数字严格增大的子序列。我们需要找到最长的递增子序列的长度（顶点数）。例如：

{6,1,8,5,3,6,9,7}的最长递增子序列为{1,3,6,9}，长度为4

问题分析：
为了更好地理解问题的解空间，我们构造一个有向无环图(dag)：
为每个元素$a_i$构造一个顶点$i$, 并建立一条边｛$i$,$j$｝，且${a_i}\le{a_j}$, ${i}\lt{j}$

所以问题转变为求解有向无环图 DAG(V,E) 的最长路径:

  1. 首先，我们定义$L(j)$为最长路径的长度（即结束于j的最长路径上的顶点数）
  2. 通过数学归纳法，假设已知$L(i)$（$i$为$j$ 的父节点）,只要找到$L(i)$的最大值, $L(j)$就等于 $max\{ L(i):(i,j)\in E \} + 1$
  3. 对于$j = 1,2,…,n$, 返回最大值

3. 编辑距离(Edit distance)

在机器翻译中，当拼写检查器遇到一个可能的单词拼写错误时，它就会在字典里寻找其他近似的单词，这种情况下，怎么根据近似度找出最合适的单词？
一种衡量两个字符串距离的方法是比较它们对齐或匹配的程度，比如 SNOWY 和 SUNNY:

"--" 表示“间断(gap)”，任何数量的”gap”可以插入到字符串的任何位置, cost 表示字母不同的列数，而编辑距离就是最小的 cost 的值。比如SNOWY 和 SUNNY 的编辑距离为3。
编辑距离之所以如此命名，是因为它表示了将一个字符串转换为另一个字符串时，字符编辑操作(删除，插入，替换)的最小数，本例中，编辑距离显示三个操作：插入 U， 替换 O –> N, 删除 W。

动态规划解法

当解决一个动态规划问题时，最关键的问题是，它的子问题是什么？只要找到它，就可以通过数学归纳的思想来解决：按照递增的顺序，迭代地解决子问题。

OK, 我们的目标是找到两个字符串 $x[1…m]$ 和 $y[1…n]$ 的编辑距离，怎么定义一个好的子问题，它应该是解决整个问题的一部分的方法。好，我们不妨来计算两个字符串的前缀部分 $x[1…i]$ 和 $y[1…j]$ 的编辑距离，称之为 $E(i,j)$, 我们的最终目标就是计算出 $E(m,n)$。

接着，我们需要知道怎么把 $E(i,j)$ 划分成更小的子问题：
首先考虑 $x[1…i]$ 和 $y[1…j]$ 的最右列的排列情况，可分为以下三种：

第一种情况的列的 cost 为1，剩余对齐的部分为 $x[1…i-1]$ 和 $y[1…j]$ ，同时也是子问题 $E(i-1,j)$
第二种类似，剩余对齐部分的子问题为 $E(i,j-1)$
第三种，最右列的 cost 为 1 ($x[i]\neq y[j]$) 或 0 ($x[i] = y[j]$), 剩余部分的子问题表示为 $E(i-1,j-1)$
综上，$E(i,j)$ 表示为：

$$E(i,j) = min\{1+E(i-1,j), 1+E(i,j-1), diff(i,j)+E(i-1,j-1)\}$$

方便起见，定义 $diff(i,j)$ 输出 1（$x[i]\neq y[j]$） 或 0（$x[i] = y[j]$）。

4. 潜在的DAG结构

每个动态规划问题都有一个 底层的DAG(有向无环图) 结构：

想象一下，每个图节点代表了一个子问题，每条边对应了解决子问题之间的一个优先级限制条件，比如节点 $v_1,…,v_k$ 指向节点 $n$，意味着“子问题 $n$ 只能在求得 $v_1,…,v_k$ 之后解出”。

在编辑距离的例子中，节点对应了图表中的位置 $(i,j)$, 边 $(i-1,j) \to (i,j),\quad (i,j-1) \to (i,j),\quad (i-1,j-1) \to (i,j)$ 显示优先级。
更进一步，我们可以给每条边加上权重，就能够计算出DAG的最短路径(参考Bellman-Ford算法)。比如，把所有边 $\{(i-1,j-1) \to (i,j) : x[i]=y[j]\}$ 权重（cost）设为 0，其余边设为 1，最终我们要求出从节点 $s(0,0)$ 到 $t(m,n)$ 的最短路径。如图所示：图中显示路径向下为删除，向右为插入，对角为替换($x[i] \neq y[j]$) 或匹配（$x[i]=y[j]$,权重为 0）。
